Nosso WorkFlow

Entendimento

Ao iniciar um projeto é preciso entender as expectativas do cliente e descobrir as necessidades do usuário. Para isso utilizamos técnicas que exploram as informações de todos os envolvidos no processo.


"Persona de um analista de requisitos."

Esse é o caso das entrevistas com stakeholders, por exemplo. Através delas podemos ouvir desejos e necessidades das pessoas-chave para, assim, garantir a qualidade desejada do produto.
O uso de personas também é fundamental pois, uma vez mapeado o público-alvo, é possível utilizar de estratégias de usabilidade mais específicas para aquele nicho de mercado. Dessa forma entregamos um produto final mais completo e adequado a todos.

Atividades

  • Entrevistas com stakeholders;
  • Mapeamento de objetivos e necessidades;
  • Definição de personas/mapa de empatia e user stories;

Participam do processo

  • Flow
  • Cliente
  • Usuário

Entregáveis

  • Roadmap de funcionalidades
  • Personas / mapa de empatia;
  • User stories;


"Mapa de empatia desenvolvido para a persona de analista de requisitos."

Pesquisa

Com as necessidades mapeadas começamos a alinhar o projeto ao ponto de vista mercadológico, analisando, não só concorrentes, como também as melhores e mais recentes soluções de design propostas no mercado atual.


"Modelo de análise competitiva Flow."

Nessa etapa são coletadas referências do que está sendo feito em outros projetos do mesmo nicho, a fim de garantir que o público alvo já comece tendo familiaridade com o produto. Também é feito um apanhado das últimas tendências de design e usabilidade de acordo com os requisitos do projeto, direcionando a criação para as melhores soluções nas próximas etapas.

Atividades

  • Análise competitiva
  • Referências de arquitetura de informação
  • UX trends

Participam do processo

  • Flow
  • Cliente (opcional)

Entregáveis

  • Mapa de análise competitiva

Prototipação

Depois de entender todos os requisitos do projeto chegou a hora de propor ideias. A prototipação, começada no papel e evoluída para o digital, traz as primeiras formas concretas do projeto com as melhores soluções.
Os primeiros rascunhos em papel são utilizados como ponto de partida para discutir e estruturar as funções das telas, permitindo assim a visualização das ideias de forma rápida e prática.
Quando passamos para o protótipo digital navegável fica ainda mais fácil entender como os fluxos interagem entre si e como o usuário pode percorrer entre eles, já que se trata de um modelo interativo, que permite navegar as telas antes de serem desenvolvidas.

Atividades

  • Exploração das ideias de layout e casos de uso do produto
  • Análise e discussão das propostas com os envolvidos

Participam do processo

  • Flow
  • Cliente

Entregáveis

  • Protótipos em papel
  • Protótipos digitais navegáveis


"Os protótipos em papel e digitais do site do Mestre do Erp"

Validação

Feito o Protótipo é necessário confirmar que o que foi proposto está de acordo com as expectativas de todas as pessoas envolvidas com o projeto.
Com o cliente a validação é feita durante a criação, acompanhando todas as ideias propostas e participando junto. Para avaliar a usabilidade para os usuários, a Flow utiliza metodologias de validação ágeis e diretas que rapidamente mostram pontos fracos que o produto final poderia apresentar caso essa etapa não fosse realizada. Entre esses métodos, temos o PURE (Pragmatic Usability Rating by Experts) que quantifica em números a facilidade de uso de um determinado produto, tudo isso com o objetivo de garantir uma entrega mais adequada para todos.

Atividades

  • Método PURE
  • Ajustes e documentação

Participam do processo

  • Flow
  • Usuário (opcional)

Entregáveis

  • Relatório Método PURE


"Resultados de um teste de análise PURE desenvolvido pela Flow."

Visual Design

Em seguida é preciso dar identidade ao projeto. Isso é feito na etapa de visual design que, além de auxiliar em boas práticas de usabilidade, uma das formas de garantir uma maior fidelização de clientes durante o primeiro contato com o seu produto.
Aqui é definida a estética do projeto, quais serão as cores, fontes, ícones e estilos de todos os componentes. Assim que a identidade é aprovada, passa a ser aplicada em todas as telas, buscando garantir harmonia e consistência entre todas as informações.


"Criação de Guidelines de visual design com o Style Guide padrão da Flow."

Nessa etapa também é feita a documentação para o desenvolvimento, garantindo que o usuário final receba seja exatamente o que foi desenhado anteriormente."

Atividades

  • Design de interface
  • Protótipo digital e navegável
  • Motion design
  • Criação de guidelines / assets
  • Documentação para desenvolvimento
  • Avaliação de heurística de usabilidade

Participam do processo

  • Flow
  • Cliente

Entregáveis

  • Protótipos digitais navegáveis
  • Style Guide
  • Documentação Zeplin


"Visual Design do nosso projeto para a IVAN — Negócios Imobiliários"

Desenvolvimento

Finalizadas as etapas de criação, passamos para o desenvolvimento, onde seu produto será programado com base no modelo de visual design.

Para garantirmos que o que foi definido em protótipo seja devidamente cumprido na versão programada nós mesmos fazemos a codificação front-end no caso de projetos web, e fazemos um relatório de análise comparativa no caso de sistemas e aplicativos que são desenvolvidos por terceiros.

Atividades

  • Desenvolvimento frontend (HTML5, CSS e Wordpress)
  • Análise de qualidade da implementação do frontend)

Participam do processo

  • Flow
  • Equipe de desenvolvimento

Entregáveis

  • Projeto completo e funcional (em caso de projetos web)
  • Relatório de revisão (em caso de sistemas e aplicativos)


"Relatório de revisão para o site principal da Softbox."

Lançamento

A última etapa do processo da Flow é o lançamento do seu produto. Com ele no mercado conseguimos obter dados reais de interação com o usuário e, assim, coletar e avaliar feedbacks e métricas de uso. Dessa forma é possível identificar falhas e sugerir melhorias, para garantir que você sempre obtenha o melhor resultado possível do seu produto.

Atividades

  • Coletar feedbacks dos usuários e analisar a forma como o produto está sendo usado

Participam do processo

  • Flow
  • Cliente
  • Usuário

Entregáveis

  • Users feedbacks
  • Métricas de uso
  • Relatório de Análises e decisões de melhorias


"Heatmap, um dos nossos métodos de avaliação de métricas de uso."
